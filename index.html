Features
StripeSee it here!
  • Main focus of contributions when at Stripe. Distributed system written using Temporal in Scala. The Kafka Control Plane is the service that we use to automate maintenance operations for our kafka clusters. For example, operations such as cluster rebalancing, broker replacements, and cluster rolling restarts.
  • Contributed topic configuration management workflow to Kafka Control Plane that manages all topic configurations for all users, across all Kafka clusters at Stripe
  • workflow coordination mechanism that prevents multiple cluster operations from being applied to a Kafka cluster at the same time. I did this by writing a locking mechanism that leverages Temporal’s mutual exclusion Workflow Id principals. Whenever an operation is to be run on a Kafka cluster, a LockWorkflow is spawned that has the name of the kafka cluster it is operating on. So for example, if you have a kafka cluster name kafka-00 and you want to replace a broker on said cluster, a LockWorkflow must be spawned with the cluster name. If there is a lock that is already acquired on the cluster, then the operation trying to acquire the lock will spin until it can acquire the lock, and proceed as normal. Ultimately, this doesn’t help with providing ordering and priority to specific operations, but it prevents multiple operations being applied to a cluster at once that could result in dire states, such as under the minimum number of in-sync replicas.
Research
Efficient Truncated Regression with Unknown Noise Variance - arxiv
  • Selected to NeurIPs 2021
  • we provide the first computationally and statistically efficient estimators for truncated linear regression when the noise variance is unknown, estimating both the linear model and the variance of the noise. Our estimator is based on an efficient implementation of Projected Stochastic Gradient Descent on the negative log-likelihood of the truncated sample. Importantly, we show that the error of our estimates is asymptotically normal, and we use this to provide explicit confidence regions for our estimates
  • Conducted experiments no both synthetic and semi-synthetic data to show the algorithm's performance in practice; see github
